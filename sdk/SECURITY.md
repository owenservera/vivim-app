# Security Policy

## Supported Versions

We only provide security updates for the latest stable version of the VIVIM SDK.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within the VIVIM SDK, please follow these steps:

1. **Do not** open a public issue.
2. Send an email to security@vivim.net (or use our bug reporting system if available).
3. Include as much detail as possible, including steps to reproduce.

We will acknowledge your report within 48 hours and provide a timeline for a fix if applicable.

## Responsible Disclosure

We ask that you follow responsible disclosure principles:
- Give us reasonable time to investigate and mitigate the issue before making it public.
- Do not exploit the vulnerability for any purpose.
- Do not perform DDoS attacks or social engineering.

Thank you for helping us keep the VIVIM ecosystem secure!
